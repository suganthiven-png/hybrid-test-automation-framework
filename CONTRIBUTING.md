# Contributing to hybrid-test-automation-framework
Thanks for your interest in contributing! This project follows a simple branching and PR workflow.

How to contribute
1. Fork the repository on GitHub (if you're not a direct collaborator).
2. Create a new branch for your work:
```cmd
git checkout -b feature/short-description
```
3. Make small, focused commits with clear messages.
4. Run tests locally and ensure everything passes.
5. Push your branch and open a Pull Request.

PR checklist
- [ ] Code builds and tests pass locally.
- [ ] Code is well-formatted and follows project style.
- [ ] New features include or update tests where applicable.
- [ ] Update README or other documentation if necessary.

Coding standards
- Prefer small, focused commits.
- Keep public APIs stable. If breaking changes are required, discuss in PR description.

Reporting issues
- Open an issue on GitHub including: steps to reproduce, expected behavior, actual behavior, logs/stack traces if available.

Contact
- Mention team members or maintainers in PRs for review.
